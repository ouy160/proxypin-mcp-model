# REQ-20260611-01 依赖分析报告

## 任务依赖图

```
T-01 (extension.dart 添加 AKI/SKI OID)
  │
  ├──> T-02 (实现 SKI 计算)
  │      │
  │      └──> T-03 (generateSelfSignedCertificate 添加 AKI/SKI 扩展)
  │             │
  │             ├──> T-05 (cert_data.dart 添加 AKI/SKI 字段)
  │             │      │
  │             │      └──> T-06 (单元测试)
  │             │
  │             └──> T-04 (_getExtensionsFromSeq 解析 AKI/SKI)
  │                    │
  │                    └──> T-06
  │
  └──> T-07 (集成测试) 需等待 T-06
```

## 串行执行顺序

```
T-01 → T-02 → T-03 → T-05 → T-04 → T-06 → T-07
```

## 依赖说明

| 上游 | 下游 | 依赖原因 |
|---|---|---|
| T-01 | T-02, T-03, T-04 | 需要 OID 常量才能在 ASN.1 编码/解析中引用 |
| T-02 | T-03 | T-03 中 AKI 字段值需要 T-02 计算的 SKI |
| T-03 | T-05, T-04 | 证书生成后才能定义数据模型和解析逻辑 |
| T-05 | T-06 | 数据模型确定后单元测试才能断言 |
| T-04 | T-06 | 解析逻辑完成后单元测试才能验证 |
| T-06 | T-07 | 单元测试通过后才能进行集成验证 |

## 并行可能性

理论上 T-04（解析）和 T-05（数据模型）可以并行，但实际：
- T-05 依赖 T-03 的输出（先有生成才能定义模型）
- T-04 依赖 T-05 的模型
- 串行更简单清晰

## 串行模式决策

接口数 = 0，前端 = 0，后端 = 1（单 Dart 模块）
**决策：串行模式**
